# GMTerminal

GMTerminal позволяет в один клик устанавливать приложения, выдавать необходимые разрешения, а так же выполнять любые ADB команды без ноутбука. Поддерживается быстрый переход в Инженерное меню или Настройки Android.

> Если ADB соединение не устанавливается — подключите ноутбук к устройству/машине и выполните `adb tcpip 5555`.

## Установка

В этом разделе доступно добавление источников APK файлов и установка приложений.

- Добавление пользовательских источников через URL (поддерживается Яндекс Диск)
- Выбор нескольких APK для пакетной установки
- Привязка файлов данных к APK, как например, аудиокодеки для MX Player
- Отчёт по результатам установки с деталями ошибок
- Повтор неудавшихся установок

> При наличии `permissions.json` в папке с APK файлами, приложение автоматически применяет необходимые разрешения после установки APK.

## Разрешения

Управление разрешениями установленных приложений.

- Поиск по установленным приложениям
- Быстрые действия из выпадающего меню:
  - Автоматически выдать все разрешения
  - Разрешить `SYSTEM_ALERT_WINDOW`
  - Разрешить `MANAGE_EXTERNAL_STORAGE`
  - Разрешить `REQUEST_INSTALL_PACKAGES`
  - Добавить в белый список (deviceidle)
  - Удалить приложение
- Возможность добавить пользовательские действия — произвольные ADB команды
- Лог выполнения команд в реальном времени
- Долгое нажатие на приложение — скопировать имя пакета

## Терминал

Выполнение любых ADB команд прямо на устройстве.

- История команд (до 100 записей), навигация стрелками вверх/вниз
- Избранные команды: добавление, удаление, выбор из списка с возможностью редактирования
- Копирование вывода в буфер обмена
- Отмена выполняющейся команды

## Настройки

- **ADB Хост / Порт** — адрес и порт целевого устройства
- **Подключаться при запуске** — автоматически подключаться к ADB при открытии приложения
- **Скрывать клавиатуру при выполнении** — скрывать клавиатуру при нажатии кнопки «Выполнить» на вкладке «Терминал»
- **Использовать системный установщик при обновлении** — обновлять приложение через стандартный диалог системы вместо ADB
- **Избранные команды** — управление списком избранных ADB команд
- **Шаблоны команд** — управление пользовательскими действиями
- **Глобальный файл разрешений** — скачать или обновить файл разрешений; кнопка ℹ показывает список приложений в базе; кнопка GitHub открывает репозиторий с исходным файлом
- **Масштаб интерфейса** — 6 уровней от 100% до 150%

## Глобальный файл разрешений

### Что это

Глобальный файл разрешений — набор разрешений, которые нужно выдать после установки приложения. Применяется автоматически после установки приложения на вкладке «Установка» или вручную, для уже установленных приложений, на вкладке «Разрешения».

### Откуда берётся

```
https://raw.githubusercontent.com/ChebanovDD/GMTerminal/main/permissions.json
```

### Структура файла

```json
{
  "Название приложения": {
    "package": "com.example.app",
    "appops": ["SYSTEM_ALERT_WINDOW", "MANAGE_EXTERNAL_STORAGE"],
    "pm_grants": ["android.permission.READ_CONTACTS", "android.permission.CAMERA"],
    "deviceidle_whitelist": true,
    "accessibility_services": ["com.example.app/.MyAccessibilityService"],
    "notification_listeners": ["com.example.app/.MyNotificationListener"]
  }
}
```

#### Поля объекта приложения

| Поле | Тип | Описание |
|------|-----|----------|
| `package` | `string` | Имя пакета Android |
| `appops` | `string[]` | Операции AppOps для установки через `appops set` |
| `pm_grants` | `string[]` | Разрешения для выдачи через `pm grant` |
| `deviceidle_whitelist` | `boolean` | Добавить в whitelist (не ограничивать в фоне) |
| `accessibility_services` | `string[]` | Компоненты служб доступности для включения |
| `notification_listeners` | `string[]` | Компоненты слушателей уведомлений для регистрации |

Все поля необязательны — можно указать только нужные.

### Порядок применения разрешений

При нажатии «Выдать все разрешения» команды выполняются в следующем порядке:

1. **AppOps** — `appops set <package> <op> allow`
2. **pm grant** — `pm grant <package> <permission>`
3. **Doze whitelist** — `dumpsys deviceidle whitelist +<package>`
4. **Accessibility services** — объединяет текущий список с новыми сервисами через `settings put secure enabled_accessibility_services`
5. **Notification listeners** — аналогично через `settings put secure enabled_notification_listeners`

### Как использовать

1. Перейти на экран «Разрешения»
2. Найти нужное приложение в списке
3. Выбрать «Автоматически выдать разрешения» из меню действий

> При установке приложений на вкладке «Установка», разрешения выдаются автоматически.

## Локальный файл разрешении

Локальный файл разрешений привязан к конкретному источнику APK (пользовательскому или встроенному). Файл `permissions.json` размещается рядом с APK в источнике и автоматически загружается при работе с этим источником. Формат файла идентичен глобальному.

При установке APK из источника с локальным файлом разрешений — приложение предлагает автоматически применить разрешения сразу после установки.

> Локальный файл разрешений имеет приоритет над глобальным. Если локальный файл разрешений не найден, будут применены разрешения из глобального файла.
